# 메모리계층(memory hieraarchy)
레지스터, 캐시, 주기억장치, 보조기억장치로 구성
- 레지스터 : CPU 내의 작은 메모리, 휘발성, 속도 가장 빠름, 기억 용량이 가장 적음
- 캐시 : CPU내의 L1, L2 캐시를 지칭, 휘발성, 속도빠름, 기억용량이 적음
- 주기억장치 : RAM을 가리키며, 휘발성, 속도 보통, 기억 용량이 보통
- 보조기억장치 : HDD, SDD를 일컬으며 비휘발성, 속도 낮음, 기억 용량이 많음

#### 계층이 존재하는 이유
1. **더 빠른 접근과 처리속도가 증가** 
- 우리는 보통 많이 쓰는 것을 다시 많으 씀. 이 때문에 특정 데이터에 많이 접근하게 되는데 좀 더 작은 캐시 메모리에 해당 데이터가 있다면 더 빠르게 해당 데이터에 접근이 가능. 그로인해 처리 속도도 증가하게 됨  

**2. 비용의 효율성**
- 캐시메모리는 비싸고 램 등아래로 갈수록 비용은 더 저렴. 계층이 있고 캐싱 때문에 비용을 좀 더 효율적으로 쓸 수 있음

**3. 자원의 효율적 사용**
- 메모리 계층 구조는 자주 접근하는 데이터는 빠른 메모리에, 덜 접근하는 데이터는 느린 메모리에 저장하여 자원을 효율적으로 사용할 수 있음.
이렇게 하면 거의 접근하지 않은 데이터에 비싸고 빠른 메모리를 사용하지 않게 되어 자원을 낭비하지 않게 됨.
우리가 흔히 볼 수 있는 로딩중이라는 것은 사실 이 내부에서 하드디스크 또는 인터넷에서 데이터를 읽어 RAM으로 전송하는 과정이 아직 끝나지 않았음을 의미

# 가상메모리, 페이지테이블, 페이지폴트, 스레싱
#### 가상 메모리(virtual memory)란?
컴퓨터가 실제로 이용가능한 메모리 자원을 추상화하여 이를 사용하는 사용자에게 매우 큰 메모리로 보이게 만드는 메모리 관리 기법. 
이는 그냥 커보이게만 하는게 아니라 그 안에 들어가 있는 페이지 테이블, MMU, TLB, 페이지 폴트 등이 어우러진 시스템을 통틀어서 말함

**[참고]**
가상주소는 MMU와 페이지테이블에 의해 실제주소로 변환
MMU는 CPU와 메모리 사이에 위치한 하드웨어 컴포넌트로, 가상 주소와 물리 주소 사이의 변환(주소 변환)을 담당. 프로그램이 가상 주소로 메모리에 접근하면, MMU는 이를 페이지 테이블에 기반해 실제 물리 주소로 변환하여 올바른 메모리 위치에 접근할 수 있도록 도움
페이지 : 가상 메모리를 사용하는 최소 크기 단위
프레임 : 실제 메모리를 사용하는 최소 크기 단위
![](https://velog.velcdn.com/images/shypanda0119/post/80900d4f-8424-4f88-9704-52e0a499b4ab/image.png)

#### 페이지테이블
가상 메모리는 가상주소와 실제주소가 매핑되어있는 자료구조인 페이지 테이블로 관리되며 이 때 속도 향상을 위해 캐싱계층인 TLB를 사용
가상주소에서 바로 페이지테이블을 가는게 아니라 TLB에서 잇는지를 확인하고 만약 없다면 페이지테이블로 가서 실제주소를 가져옴
![](https://velog.velcdn.com/images/shypanda0119/post/900a8cce-1865-40c4-916a-4727a9af4e7b/image.png)

### 프로세스A와 프로세스B가 실행 및 종료시의 메모리 할당 과정
예를 들어 프로세스A와 프로세스B가 실행됐을 때 메모리 할당 등을 알아보면서 가상메모리와 물리메모리를 살펴보자.

#### 가상 메모리와 물리 메모리
각 프로세스는 자신만의 가상 주소 공간(Virtual Address Space) 을 가짐.
가상 주소는 프로세스가 메모리에 접근할 때 사용하는 주소이며, 다른 프로세스와 독립적
예를 들어,
프로세스 A → 가상 주소 0x0000
프로세스 B → 가상 주소 0x1000
라고 가정

물리 메모리는 실제 RAM의 주소 공간이며, 가상 주소와 직접적인 관계는 없음.
이 가상 주소를 실제 물리 주소로 변환해주는 것이 페이지 테이블(Page Table) 이고,
CPU 내부의 MMU(Memory Management Unit) 가 이를 기반으로 가상 주소를 물리 주소로 변환한다.

즉, 가상 주소 → 페이지 테이블 → 물리 주소의 과정을 거침

#### 프로세스 처음 실행 시
프로세스 A와 B가 처음 실행되었다고 가정

프로세스 A
가상 주소: 0x0000
물리 주소: 0x2000에 매핑

프로세스 B
가상 주소: 0x1000
물리 주소: 0x3000에 매핑

이처럼 서로 다른 물리 메모리 공간에 매핑되므로 충돌이 발생하지 않음.

#### 프로세스 종료
프로세스 A와 B가 종료되면,
각 프로세스의 가상 주소 공간이 해제되고 페이지 테이블이 제거되며 사용 중이던 물리 메모리도 운영체제에 의해 반환.

#### 종료 후 재실행
A와 B가 다시 실행되면 운영체제는 새로운 가상 주소 공간을 할당.
이때 가상 주소는 이전과 동일할 수도 있고, 다를 수도 있음.

예를 들어,

프로세스 A
가상 주소: 0x0000
물리 주소: 0x4000에 매핑될 수 있음

프로세스 B
가상 주소: 0x1000
물리 주소: 0x5000에 매핑될 수 있음

**중요한 점은 가상 주소가 동일하더라도 실제 물리 주소는 달라질 수 있다는 것**

#### 충돌이 발생하지 않는 이유?
각 프로세스는 독립적인 페이지 테이블을 가지고, MMU는 현재 실행 중인 프로세스의 페이지 테이블을 기준으로 주소를 변환.

A의 0x0000
B의 0x0000
은 서로 다른 물리 주소로 매핑될 수 있으며,
가상 주소는 해당 프로세스 내부에서만 유효한 값이기 때문에 충돌이 발생하지 않음

#### 페이지폴트란?
프로세스가 참조하려는 주소가 가리키는 페이지가 현재 물리 메모리(RAM)에 없을 때 발생하는 현상
가상메모리는 작은 메모리를 매우 큰 메모리로 보이게끔 하는 것이기 때문에 참조하려는 메모리 영역이 상대적으로 작은 실제 메모리에 없을 수도 있음.
이 때 페이지폴트가 일어남
![](https://velog.velcdn.com/images/shypanda0119/post/d6551b6e-800d-4d26-9cb9-3764c2bb659b/image.png)

![](https://velog.velcdn.com/images/shypanda0119/post/281ac041-ea86-4ce9-ae79-9d5a465559f7/image.png)
또한, 운영체제는 전체 프로세스의 메모리 내용을 디스크로부터 한 번에 RAM에 올리지 않음. 처음에는 필요한 부분들만 효율적으로 올리기 때문에 이런 현상이 일어남

#### 페이지 폴트 이후의 과정
1. 페이지폴트 이후 OS에게 트랩을 전송. OS는 먼저 페이지 폴트가 유효한 접근인지 점검. 만약 비정상적인 접근이라면 프로세스를 강제 종료할 수 있음.
2. OS는 물리 메모리에서 빈 프레임이 있는지 확인하고 다음의 과정이 일어남
	1) 빈 프레임이 있을 경우 : 디스크에서 필요한 페이지를 바로 메모리에 가져와 		적재(Swap-in). 이때 기존 페이지를 내보내는 작업(Swap-out)은 없음.
    2) 빈 프레임이 없을 경우 : 페이지 교체 알고리즘(Page Replacement 			   Algorithm)을 수행하여 기존의 한 페이지를 선택하고 디스크로 내보냄(Swap-		 out), 빈 프레임을 확보한 뒤 필요한 페이지를 디스크에서 메모리로 가져옴.
![](https://velog.velcdn.com/images/shypanda0119/post/befddc04-073d-4918-8365-d46b3ca32f87/image.png)

**[참고]**
이때 디스크 I/O 작업은 CPU 처리 속도에 비해 매우 느리므로, 페이지 폴트가 자주 발생할수록 CPU는 작업을 하지 못하고 대기 시간이증가하여 CPU 이용률이 저하됨
- 메모리 접근 시간 : 수백 나노초(ns) 수준
- 디스크 접근 시간 : 수 밀리초(ms) 수준(수백만 나노초)

**[참고]**
운영체제에서 스와핑은 일반적으로 디스크와 메모리 간에 데이터를 교환하는 전체 과정을 의미하며 아래 두 과정을 합쳐 스와핑으로 부르거나 각각 스와핑으로 부르기도 함
- Swap-in : 디스크 -> 메모리
- Swap-out : 메모리 -> 디스크

**[참고]**
물리 메모리가 부족할 때, 사용하지 않는 메모리 페이지를 하드 디스크의 특정 영역으로 옮겨 저장하는 공간을 스왑 영역이라고 부름

### 스레싱이란?
실제 프로세스 실행보다 스와핑 작업에 대부분의 시간이 소요되는 현상을 말하며, 페이지 폴트가 빈번하게 발생하면, 필요한 데이터를 디스크에서 메모리를 불러오는 과정이 잦아지고 이로 인해 스와핑이 과도하게 일어남

그 결과, CPU는 실제 연산을 수행하기보다 스와핑 대기 상태에 머무르게 되어 전체적인 CPU 이용률이 낮아짐

이러한 상황에서 운영체제는 CPU의 낮은 사용률을 보완하고자 추가 프로세스를 메모리에 적재하게 되는데, 이는 이미 부족한 메모리 자원을 더욱 압박하게 됨. 결국, 메모리 부족과 빈번한 페이지 폴트가 서로 악화시키는 악순환이 발생하고, 이로 인해 시스템 전체의 성능이 급격히 저하되는 스레싱 현상이 나타남.

![](https://velog.velcdn.com/images/shypanda0119/post/aae56af5-2791-410f-8f59-5972f3bf3121/image.png)
이를 하드웨어적으로 해결하는 방법은 메모리를 늘리거나 HDD를 사용한다면 SSD로 바꾸는 방법이 있으며 운영체제에서 해결하는 방법은 작업세트와 PFF가 있음

#### 작업세트(Working set)란?
프로세스의 과거 사용이력을 기반으로 많이 사용하는 페이지집합을 만들어 한꺼번에 미리 메모리에 로드하는 것을 말함
프로세스가 자주 접근하는 페이지들이 이미 메모리에 존재하게 되면 프로세스가 해당 페이지에 접근할 때 디스크에서 데이터를 불러오는 페이지 폴트(page fault)가 발생할 가능성이 줄어듬

#### PFF(Page Fault Frequency)란? 
각 프로세스의 페이지 폴트 발생 빈도를 모니터링하여, 메모리 프레임의 할당량을 동적으로 조절하는 방법

**- 상한선 초과**
프로세스의 페이지 폴트 빈도가 상한선에 도달하면, 해당 프로세스에 더 많은 프레임을 할당하여 작업세트를 확장

**- 하한선 미달**
프로세스의 페이지 폴트 빈도가 하한선에 도달하면, 운영체제는 이 경우 일부 프레임을 회수하여 다른 프로세스에 재할당할 수 있음

### 가상 메모리의 장점
#### 메모리 효율적 사용
가상 메모리는 프로그램 실행 시 모든 데이터를 한꺼번에 메모리에 적재하지 않고, 실제로 필요한 시점에 해당 메모리 페이지만 동적으로 불러옴. 이 덕분에 제한된 RAM 내에서도 대용량 프로그램을 효율적으로 실행할 수 있으며, 이러한 방식을 Demand Paging(요구 페이징)이라고 함

#### 메모리 과잉 할당
운영체제는 각 프로세스에 가상 주소 공간을 넉넉하게 할당하여 실제 RAM보다 훨씬 큰 주소 공간을 사용할 수 있어서 메모리 자원을 최대한 효율적으로 배분할 수 있음

#### 프로세스 간 메모리 보호
각 프로세스는 독립적인 가상 주소 공간을 갖고, 동일한 가상 주소라도 서로 다른 물리 메모리 영역에 매핑. 이를 통해 오류나 데이터 충돌이 한 프로세스에 국한되어 다른 프로세스에 영향을 주지 않고 특정 프로세스의 악의적 접근을 방지할 수 있음

#### 메모리 접근 제어
가상 메모리는 메모리를 일정한 크기의 페이지로 나누어 관리하는데 이 때 읽기, 쓰기, 실행 권한을 설정할 수 있음
-> 권한 없는 접근 시도가 즉시 감지되고 차단되어, 버퍼 오버플로우나 코드 인젝션 같은 보안 위협을 예방
