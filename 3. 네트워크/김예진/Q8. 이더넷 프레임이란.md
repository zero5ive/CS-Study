# 이더넷 프레임

데이터 링크 계층(L2)의 데이터 단위(PDU)이며, 물리적인 네트워크 매체를 통해 데이터를 전송하는 가장 기본이 되는 틀이다.

## 프레임 구조

| 필드명 | 크기 | 설명 |
| --- | --- | --- |
| **Preamble** | 7 Bytes | 송수신 간의 **비트 동기화(Clock Sync)**를 위해 사용되는 101010... 패턴 |
| **SFD (Start Frame Delimiter)** | 1 Byte | `10101011` 비트 패턴으로, 진짜 프레임 시작을 알리는 부분 |
| **Destination Address** | 6 Bytes | 수신 측의 **MAC 주소** |
| **Source Address** | 6 Bytes | 송신 측의 **MAC 주소** |
| **EtherType (Type/Length)** | 2 Bytes | 상위 계층 프로토콜 종류를 식별 (예: `0x0800`은 IPv4, `0x86DD`는 IPv6) |
| **Payload** | 46~1500 Bytes | 상위 계층(IP 패킷 등)에서 내려온 실제 데이터 |
| **FCS (Frame Check Sequence)** | 4 Bytes | 에러 검출용 필드. **CRC(Cyclic Redundancy Check)** 알고리즘을 사용 |
