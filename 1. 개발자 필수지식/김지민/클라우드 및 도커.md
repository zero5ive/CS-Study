# 클라우드란?
 클라우드 서비스는 내가 아닌 다른 회사의 공급자가 호스팅하고 인터넷을 통해 사용자가 제공되는 인프라, 플랫폼 또는 소프트웨어.
 
####  전통적 배포방식
 물리적인 컴퓨터 한 대에 하나의 OS를 깔고 여러 가지 프로그램을 설치하는 방식. 

#### 가상화 배포방식
 가상머신을 기반으로 배포하는 것. 가상머신이란? 컴퓨터의 하드웨어를 소프트웨어적으로 구현하는 것
 
#### 오프프레미스(off-premise)
 서버를 외부(클라우드)에 두고 사용하는 방식

#### 온프레미스(on-premise)
 서버리르 내부에 두고 사용하는 방식
 

#### laaS(Infrastructure-as-a-Service)이란?
 인프라형 클라우드서비스, 클라우드가 단지 인프라를 제공(예를 들면
 Node.js, MongoDB 등을 개발자가 직접 설치해야 하는 대신 특정 서비스에 종속되지 않음)
 
#### Paas(Platform-as-a-Service)이란?
 플랫폼형 클라우드 서비스, 클라우드가 플랫폼을 제공(예를 들면
 Node.js, MongoDB 등이 설치되어있으며 그저 클릭을 통해 해당 서비스를 이용할 수 있음. 모니터링, CI/CD가 제공)
 
#### Saas(Software-as-a-Service)이란?
 서비스형 클라우드 서비스, 완전한 서비스를 클라우드 서비스로부터 제공받아 사용.
 
#### laaS VS PaaS 
laaS -> 유연하며 플랫폼에 종속되지 않음, 이식성이 높음, 운영비 효율 낮음

Paas -> 유연하지 않으며 플랫폼에 종속됨, 이식성이 낮음, 운영비 효율 좋음

# 컨테이너란?
 애플리케이션이 한 컴퓨팅 환경에서 다른 컴퓨팅 환경으로 빠르고 안정적으로 실행되도록 코드와 모든 종속성을 패키징하는 소프트웨어 표준 단위
 
# 도커란?
 도커는 컨테이너 필요한 기능을 거의 모두 제공하는 플랫폼
![](https://velog.velcdn.com/images/shypanda0119/post/b3b7eb0e-60b6-4646-804c-c93dba76e4e9/image.png)
**도커파일**: 패키지, 환경변수설정 등을 기록한 파일. 이를 빌드해 도커이미지로 변환
**도커이미지**: 컨테이너 실행에 필요한 파일과 설정값, 데이터 등을 포함된 상태값이며 불변. 하나의 이미지에서 여러개의 컨테이너를 생성할 수 있으며 컨테이너의 상태와는 무관하게 이미지는 그대로 존재.
**도커컨테이너** : 컨테이너가 실행시키면 도커이미지에 설정된 프로그램, 데이터 등이 실제 컴퓨팅자원과 연결.
