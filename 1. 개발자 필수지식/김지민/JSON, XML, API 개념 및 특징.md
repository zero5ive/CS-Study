#### 1. Javascript 객체 문법
 키와 값으로 구성, {key : value}

#### 2. 데이터 + 교환형식

#### 3. 여러언어에서의 쓰임
 객체, 해시테이블, 딕셔너리 등으로 변환되어 쓰임
 
#### 4. 단순 배열, 문자열 표현

# JSON의 타입
 -수(Number)
 -문자열(String)
 -참/거짓(Boolean)
 -배열(Array)
 -객체(Object)
 -null
 **undefined, 메서드 등은 포함할 수 없음.**
 
 # JSON 직렬화, 역직렬화
 
####  JSON 직렬화
 외부의 시스템에서도 사용할 수 있도록 바이트(byte) 형태로 데이터를 변환하는 기술
 JSON.stringfy()

####  JSON 역직렬화
 바이트(byte) 형태의 데이터를 외부의 시스템에서도 사용할 수 있도록 변환하는 기술
 JSON.parse()
 
# XML(Extensible Markup Language)이란?
 마크업 형태를 쓰는 데이터교환 형식
 마크업 형태란? 마크업(markup)는 태그 등을 이용하여 문서나 데이터의 구조를 나타내는 방법(속성부여도 가능)
 
####  XML 구성
**1. 프롤로그 : 버전, 인코딩
2. 루트요소(단, 하나만)
3. 하위 요소들
**

#### HTML과 XML 비교
1. HTML의 용도는 데이터를 표시 / XML은 데이터를 저장 및 전송
2. HTML에는 미리 정의된 태그가 있지만 사용자는 XML에서 고유한 태그를 만들고 정의 가능
3. XML은 대/소문자를 구분하지만 HTML은 구분하지 않음.

#### JSON과 XML 비교
 JSON과 비교했을 때 닫힌 태그가 계속해서 들어가 때문에 JSON과 비교하면 무거움.
 
#### stiemap.xml
 sitemap.xml은 서비스 내의 모든 페이지들을 리스트업한 데이터
 사이트가 매우 크거나 서로 링크가 종속적으로 연결되지 않은 경우 크롤러가 일부 페이지를 누락하는 일이 있을때, 모든 페이지들을 크롤링할 수 있도록 함

# 인터페이스
 인터페이스는 서로 다른 두 개의 시스템, 장치 사이에 정보나 신호를 주고받는 경우의 접점이나 경계면.
 
![](https://velog.velcdn.com/images/shypanda0119/post/9311683f-92f1-4d40-8335-71c32e71dff9/image.png)
#### API의 장점
1. 제공자는 서비스의 중요한 부분을 드러내지 않아도 됨
2. 사용자는 해당 서비스가 어떻게 구현되는지 알 필요없이 필요한 정보만 받을 수 있음
3. 앱 개발 프로세스를 단순화 시키고 시간과 비용을 절약할수 있음
4. 내부 프로세스가 수정됐을 때, API를 매번 수정하는 것이 아닌 API가 수정 안되게 만들 수 있음.
5. 제공자는 데이터를 한 곳에 모을 수 있음.
