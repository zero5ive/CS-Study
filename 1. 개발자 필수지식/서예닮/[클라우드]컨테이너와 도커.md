# ✏️ 컨테이너(Container)란?

## 컨테이너의 정의
컨테이너는 **애플리케이션과 그 실행에 필요한 모든 의존성(라이브러리, 런타임, 설정 파일 등)을 하나의 패키지로 묶은 경량 가상화 기술**이다.

---

##  컨테이너의 특징

### 1. 가볍다
- 가상머신(VM)과 달리 **전체 OS를 포함하지 않음**
- 일반적으로 **수백 MB 수준**

### 2. 빠르다
- 부팅 시간이 **밀리초(ms) 단위**
- 실행 오버헤드가 적음

### 3. 이식성이 좋다
- *“한 번 빌드하면 어디서나 실행된다”*
- 개발 환경과 운영 환경의 차이를 제거

### 4. 격리성
- 각 컨테이너는 **독립적인 프로세스 공간**을 가짐
- 서로 영향을 주지 않음

---

##  컨테이너 vs 가상머신(VM)

| 구분 | 컨테이너 | 가상머신(VM) |
|------|----------|--------------|
| 크기 | 수백 MB | 수 GB |
| 부팅 시간 | 밀리초 | 초 단위 |
| 오버헤드 | 적음 | 많음 |
| OS | Host OS 공유 | 개별 OS 필요 |
| 격리 수준 | 프로세스 단위 | 완전 격리 |



# ✏️ 도커(Docker)란?

##  도커의 정의
도커(Docker)는 **컨테이너를 쉽게 만들고(Build), 배포하고(Ship), 실행(Run)할 수 있게 해주는 오픈소스 플랫폼**이다.



## 도커의 주요 구성 요소

### 1. 이미지 (Image)
- 컨테이너를 실행하기 위한 **템플릿**
- 애플리케이션 코드, 런타임, 라이브러리, 환경 변수 포함
- 하나의 이미지로 여러 컨테이너 생성 가능
- 컨테이너가 삭제되어도 이미지는 유지됨


### 2. 컨테이너 (Container)
- **이미지를 실행한 결과물**
- 독립적인 실행 환경
- 이미지로부터 생성됨

### 3. Dockerfile
- 컨테이너 이미지를 만들기 위한 **설정 파일**
- 설치할 패키지, 환경 변수, 실행 명령 등을 정의
- `docker build` 명령어로 이미지 생성


### 4. 레지스트리 (Registry)
- 도커 이미지를 저장하고 공유하는 저장소
