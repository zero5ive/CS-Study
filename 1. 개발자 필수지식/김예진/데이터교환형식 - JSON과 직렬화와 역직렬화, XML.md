# 개발자 필수 지식

## 데이터 교환 형식 #1. JSON과 직렬화 / 역직렬화 (Serialization / Deserialization)

### JSON (JavaScript Object Notation)

- **JavaScript 객체 형식**으로 구분된 **데이터 교환 형식**
- **python, JavaScript, Java 등** 여러 언어에서 데이터 교환 형식으로 쓰이며, 객체 문법 말고도 단순 배열, 문자열 표현도 가능
- 사람이 읽기 쉽고(Human-readable), 경량(Lightweight)

---

### JSON의 기본 사용법

1. **JavaScript 객체 문법**
   - `key : value`
   - 키를 **쌍따옴표**(`" "`, double quotes)로 묶는 것이 **JSON**의 규칙

2. **데이터 + 교환 형식**
   - 데이터: 실제 생활 정보와 같은 사실이나 자료 등

3. **데이터의 호환**
   - 객체, 해시테이블, 딕셔너리 등으로 변환되어 쓰이며, 언어 플랫폼에 독립적으로 쓰임

4. **데이터베이스 문법과 관련 없음**
   - 키-밸류가 아닌 데이터도 저장이 가능
   - 이는 하나의 키에 여러 개의 값 저장이 가능



### JSON의 타입

- `undefined`, `null` **등은 제외**
- **Number, String, Boolean, Array, Object, null**



### JSON 직렬화 · 역직렬화

- **직렬화**
  - 외부의 시스템에 저장할 수 있도록 **byte 배열** 형태로 데이터를 변환하는 것
  - 네트워크상에서 객체는 그대로 전송될 수 없으므로, 바이트 스트림 형태로 변환하여 전송
  - 예시  
    - `JSON.stringify(b)`

- **역직렬화**
  - byte 배열 → **자바스크립트 객체** → 내부 시스템 데이터 형식
  - 예시  
    - `JSON.parse(a)`



### 직렬화 필요성

- 객체의 영속성(Persistence) 확보
- 메모리의 객체는 프로세스가 종료되면 사라지지만, 직렬화를 통해 데이터를 파일 시스템이나 DB에 저장(영속화) 가능
- 네트워크 전송을 위해 바이트 스트림 형태로 변환하여 전송



### 직렬화 / 역직렬화 사용 시 주의점

- 데이터를 주고받는 과정에서 버전 호환성, 보안(민감 정보 노출), 그리고 직렬화 성능 등을 고려해야 함
- Java 같은 언어에서는 Serializable 인터페이스 구현 시 serialVersionUID를 통해 버전 관리를 명확히 하는 것이 중요


### JSON의 활용

- 프로그래밍 언어나 프레임워크 중에 **독립적**이고 **서로 다른 시스템 간** 데이터 교환에 용이
- 주로 **API**의 **반환 값**이나, 시스템 간 **설정 파일**에 활용



## 데이터 교환 형식 #2. XML

### XML (Extensible Markup Language)

- **마크업 언어**를 쓰는 **데이터 교환 형식**


### 마크업 (mark-up) 형태

- **태그**를 이용하여 원하는 데이터의 구별을 나타내는 방법


### 구성

1. **프롤로그**
   - 버전, 인코딩

2. **루트 요소**
   - 최상위

3. **하위 요소**


### HTML과 XML의 차이

1. HTML은 데이터를 **표시**, XML은 데이터를 **저장 및 전송**하는 용도
2. HTML에는 **이미 정의된 태그**가 있지만, XML은 **사용자가 태그를 정의** 가능
3. HTML은 **대소문자 구별**을 하지 않지만, XML은 **대소문자를 구별**


### JSON과 XML의 차이

- XML은 JSON보다 **많은 여는 태그**가 들어가 **무거움**
- **JavaScript Object**로 변환하기가 JSON보다 **많은 노력**이 필요  
  (외부 라이브러리 코드 또는 구현 필요)
- JSON은 별도의 스키마 정의 표준이 없어, 데이터의 유효성 검증을 수동으로 해야 함
- XML은 **DTD(문서 형식 정의)**나 **XSD(XML Schema Definition)**를 통해 데이터 구조와 타입을 명확히 정의하고, 자동화된 유효성 검증이 가능


### XML의 활용

- 스타일(Style)이나 매핑(Mapping) 정보를 정의하는 설정 파일의 일반적인 형태인 `stylemap.xml`으로 쓰임
- SOAP(Simple Object Access Protocol)
  - 웹 서비스에서 메시지 교환을 위해 XML을 사용
- Java 설정 파일
  - Spring Framework의 초기 버전
  - Maven의 `pom.xml` 파일 등 설정 파일을 정의
- 여러 언어에서 독립적으로 쓰임
