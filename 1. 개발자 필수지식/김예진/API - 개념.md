# API

## API (Application Programming Interface)
### 정의
- 둘 이상의 컴퓨터 프로그램이 서로 통신하기 위한 규약이자, 컴퓨터 사이에 있는 중계 계층
- 구성 요소: 프로토콜(HTTP/HTTPS), 메서드(GET/POST 등), 데이터 타입(JSON/XML 등)이 정의되어 있음

### API의 장점
1. 캡슐화와 보안
  내부 DB 구조, 서버의 상수값, 비즈니스 로직 등 중요한 정보를 노출하지 않고 필요한 데이터만 선별하여 제공 가능

2. 추상화
  사용자는 내부 구현 방식(어떤 언어로 짰는지, 어떤 알고리즘인지)을 몰라도 인터페이스만 알면 기능을 사용 가능
  
3. 유지보수 용이
  내부 로직이나 DB 구조가 변경되어도 API 규격만 유지된다면 클라이언트는 코드를 수정할 필요가 없음 (하위 호환성 유지)

4. 생산성 향상 (Open API)
  이미 만들어진 기능(지도, 결제, 로그인 등)을 Open API로 가져다 써서 개발 시간과 비용을 절약

5. 데이터 집계 및 분석
  사용자 이벤트(클릭, 검색 등)를 API 호출 시점에 집계하여 데이터를 한 곳으로 중앙 집중화 가능 (예: 베스트셀러 집계)

  
### API의 종류
1. private
  회사 내부 시스템 간 통신용
주로 해시키를 하드코딩해놓고, 이를 기반으로 서버와 서버 간 통신

2. Partner 
특정 비즈니스 파트너(협력사)에게만 공개
API Key, HTTP 헤더를 공유하여 인증

3. public
모든 사람이 사용 가능
많은 트래픽을 방지하기 위해 하루 요청 수 제한, 계정 당 개수 제한 등 Rate Limiting을 두고 관리

## 인터페이스 (Interface)
### 정의
- 서로 다른 두 개의 시스템, 장치 사이에서 정보나 신호를 주고 받는 경우의 접점 혹은 경계면
- 하드웨어(USB 포트), 사용자(UI), 소프트웨어(API) 등 모든 소통 창구

### 특징
- 내부 서버가 어떻게 구현돼있는지와 상관 없이 인터페이스를 통해 통신이 가능
- 규격화된 인터페이스는 교체 가능성을 높임 (예: 어떤 컴퓨터든 USB 인터페이스만 맞으면 연결 가능한 것과 같음)
